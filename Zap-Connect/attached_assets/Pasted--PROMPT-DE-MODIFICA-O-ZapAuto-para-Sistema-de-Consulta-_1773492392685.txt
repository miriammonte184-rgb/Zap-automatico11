### 1. WHITELIST DE NÚMEROS (NOVA FUNCIONALIDADE — OBRIGATÓRIA)

O sistema **NÃO deve responder qualquer número**. Deve responder **somente números autorizados**.

#### Implementação:
- Criar tabela SQLite `whitelist` com campos: `id`, `numero` (formato: 5511999999999), `nome`, `ativo` (0/1), `criado_em`
- Criar tela no painel: **"Números Autorizados"** com CRUD completo (adicionar, listar, ativar/desativar, excluir)
- No serviço WhatsApp (`whatsappService.js`), antes de processar qualquer mensagem recebida:
  ```
  1. Extrair o número do remetente (sem @c.us)
  2. Verificar na tabela whitelist se o número existe e está ativo
  3. Se NÃO estiver na whitelist → ignorar completamente a mensagem (não responder, apenas logar)
  4. Se estiver → prosseguir normalmente
  ```
- Logar mensagem ignorada: `⛔ Número não autorizado: [numero] — mensagem ignorada`

### 2. INTEGRAÇÃO COM PLANILHA EXCEL (NOVA FUNCIONALIDADE — CORE)

#### 2.1 Configuração da planilha no painel

Criar seção **"Planilha de Dados"** na tela de configurações com:
- Campo para upload da planilha `.xlsx`
- Campo para informar o **nome da aba** a ser lida (padrão: `SD4`)
- Botão **"Carregar / Atualizar Planilha"**
- Exibir data/hora do último carregamento
- Exibir quantidade de linhas carregadas (após filtro)
- A planilha carregada deve ser salva em `./uploads/planilha_atual.xlsx`
- Os dados filtrados e processados devem ser armazenados **em memória** (objeto/array global no Node.js) para consultas rápidas

#### 2.2 Mapeamento de colunas da aba SD4

| Coluna Excel | Índice | Campo interno | Descrição |
|---|---|---|---|
| B | 1 | `codigo_material` | Código do material |
| D | 3 | `descricao` | Descrição do material |
| F | 5 | `codigo_op` | Código da OP |
| G | 6 | `qtd_empenho` | Quantidade de empenho |
| H | 7 | `data_plano` | Data do plano (data da OP) |
| K | 10 | `status_setor` | Status/setor (usado para filtro de exclusão) |
| N | 13 | `estoque_atual` | Estoque atual |
| O | 14 | `qtd_pedido_compra` | Quantidade do pedido de compra |
| P | 15 | `data_pedido_compra` | Data de chegada do pedido de compra |

> **ATENÇÃO:** A linha 1 da planilha é cabeçalho e deve ser ignorada. Os dados começam na linha 2.

#### 2.3 Regra de filtro — Linhas que devem ser IGNORADAS

Ao carregar a planilha, ignorar (excluir do processamento) todas as linhas onde a coluna K (`status_setor`) contenha qualquer um dos seguintes textos (comparação case-insensitive, verificar se CONTÉM o texto):

```
COMERCIAL
EXPEDIÇÃO
FINALIZADA
QUALIDADE
LOGISTICA
#N/D
```

#### 2.4 Lógica de consumo do estoque por OP

Quando houver uma consulta por OP, o sistema deve verificar quais materiais estão em falta seguindo esta lógica:

```
PARA CADA material distinto da OP consultada:
  1. Obter o estoque_atual do material (coluna N)
  2. Buscar TODAS as OPs que usam esse material (em toda a planilha, não só a OP consultada)
  3. Ordenar essas OPs pela data_plano (coluna H) — da mais próxima para a mais distante
  4. Simular o consumo do estoque seguindo essa ordem:
     - OP mais próxima consome primeiro
     - OP seguinte consome o restante, e assim por diante
     - Estoque nunca fica negativo na simulação (se acabou, as próximas OPs ficam sem material)
  5. Verificar quanto de estoque sobrou para a OP consultada especificamente
  6. Se o estoque disponível para a OP < qtd_empenho da OP → material está EM FALTA
  7. Calcular a quantidade em falta: qtd_empenho - estoque_disponivel_para_op
```

### 3. MOTOR DE CONSULTA VIA WHATSAPP (SUBSTITUIR o motor de regras atual)

O sistema deve detectar automaticamente o tipo de consulta com base no texto enviado pelo usuário:

#### Detecção do tipo de consulta:

```
TIPO 1 — Consulta por OP:
  - Texto enviado contém apenas números (ex: "123456", "OP 789")
  - OU começa com "OP" seguido de números (case insensitive)
  - OU o texto encontra correspondência na coluna F (codigo_op) da planilha

TIPO 2 — Consulta por Código de Material:
  - O texto enviado corresponde a algum valor na coluna B (codigo_material) da planilha
  - Pode ser parcial (busca por "contém") ou exata

TIPO 3 — Consulta desconhecida:
  - Nenhum dos anteriores
  - Responder: "❓ Não encontrei resultado para: [texto].\n\nEnvie um *código de OP* ou *código de material* para consultar."
```

#### Resposta para CONSULTA POR OP (Tipo 1):

```
Se não houver materiais em falta:
────────────────────────────
✅ *OP [numero]*
Todos os materiais estão disponíveis!
Nenhuma falta identificada para esta OP.
────────────────────────────

Se houver materiais em falta:
────────────────────────────
⚠️ *OP [numero]* — Materiais em falta:

1️⃣ *[CODIGO_MATERIAL]* — [DESCRIÇÃO]
   📦 Falta: [qtd_em_falta] unidades
   🛒 Pedido: [qtd_pedido_compra] un.
   📅 Previsão: [data_pedido_compra]

2️⃣ *[CODIGO_MATERIAL]* — [DESCRIÇÃO]
   📦 Falta: [qtd_em_falta] unidades
   🛒 Pedido: [qtd_pedido_compra] un.
   📅 Previsão: [data_pedido_compra]

(... continua para cada material em falta)

Total de materiais em falta: [N]
────────────────────────────
```

#### Resposta para CONSULTA POR CÓDIGO DE MATERIAL (Tipo 2):

```
────────────────────────────
📦 *Material: [CODIGO_MATERIAL]*
📋 [DESCRIÇÃO COMPLETA]

📊 Estoque atual: [estoque_atual] un.
🛒 Pedido de compra: [qtd_pedido_compra] un.
📅 Previsão de chegada: [data_pedido_compra]

🏭 OPs que utilizam este material:
• OP [codigo_op] — Empenho: [qtd_empenho] — Data: [data_plano]
• OP [codigo_op] — Empenho: [qtd_empenho] — Data: [data_plano]
(máximo 10 OPs listadas, ordenadas por data)
────────────────────────────
```

> Se não encontrar o material: `❌ Material *[codigo]* não encontrado na planilha.`

### 11. EXEMPLO DE FLUXO COMPLETO (para validação)

**Cenário:** Usuário envia `789456` (código de uma OP)

```
1. Sistema recebe: "789456"

3. detectarTipoConsulta("789456") → verifica se existe como OP na planilha → retorna 'op'
4. consultarPorOP("789456"):
   a. Busca todas as linhas com codigo_op = "789456"
   b. Para cada material da OP:
      - Material: 50.08.0383
      - Estoque atual: 11
      - Todas as OPs que usam 50.08.0383 ordenadas por data_plano:
        * OP 111111 data: 16/03 empenho: 5  → consume 5  → resta: 6
        * OP 222222 data: 23/03 empenho: 4  → consume 4  → resta: 2
        * OP 789456 data: 30/03 empenho: 8  → consume 2  → FALTA 6
      - Material está em falta: faltam 6 unidades
   c. Retorna lista de materiais em falta com dados do pedido de compra
5. Formatar resposta com emojis e estrutura definida
6. Adicionar delay de 2-5s
7. Enviar resposta via WhatsApp
8. Salvar no histórico
9. Logar no painel
```
